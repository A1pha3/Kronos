## 一句话定义

**Kronos** 是一个面向金融蜡烛图（K 线）数据的开源基础模型。根据根目录 `README.md`，它通过“分词 + 自回归预测”的两阶段框架，把连续的 OHLCV 数据转换为离散令牌序列，再在令牌空间中进行生成式预测。

### 当前仓库能直接确认什么？

以下内容可以直接从仓库文档和源码确认：

1. **两阶段结构**：`KronosTokenizer` 先把连续 OHLCV 数据编码为离散令牌，`Kronos` 再预测未来令牌，最后由分词器解码回数值
2. **层级令牌**：默认配置下令牌会被拆成 `s1` 和 `s2` 两部分，预测时先生成 `s1`，再基于 `s1` 生成 `s2`
3. **高层接口**：`KronosPredictor` 将标准化、时间特征、令牌生成和反标准化封装为可直接调用的预测接口
4. **模型发布形态**：根目录 `README.md` 给出了 `mini`、`small`、`base`、`large` 等模型家族信息，其中 `large` 标注为未开源

> 根目录 `README.md` 的 News 区域还提到项目在 2025-11-10 被 AAAI 2026 接收。这里保留该仓库自述，但本文档主要关注代码与接口本身，不把会议录用状态展开成技术正确性的论证。

## 为什么需要 Kronos？

### 传统方法的局限

传统的金融预测模型通常直接处理连续的数值数据，面临两个核心困难：

1. **连续空间建模困难**：OHLCV 数据是连续的多维时间序列，直接在连续空间进行自回归预测容易产生误差累积——每一步的微小偏差会在多步预测中被放大，导致长期预测偏离
2. **缺乏通用性**：针对特定市场或品种训练的模型难以迁移到其他市场。为每只股票单独训练一个 LSTM 模型既不经济也不可扩展

### Kronos 的解决思路

Kronos 借鉴了大语言模型（LLM）的成功范式：**先将连续数据离散化为"令牌"，再在离散令牌空间进行自回归预测**。

这种设计的优势：

- **离散空间更稳定**：避免了连续值回归中的误差累积问题。即使某一步的令牌预测有偏差，不会像连续值那样"放大"
- **采样策略灵活**：可以应用温度采样、核采样等成熟的文本生成技术，控制预测的确定性和多样性
- **易于复用生成式采样机制**：源码直接提供了 `temperature`、`top_k`、`top_p`、`sample_count` 等控制项

### 这种设计的代价

没有一种架构是完美无缺的。Kronos 的两阶段设计也带来了以下代价：

| 代价 | 说明 | 缓解措施 |
|------|------|---------|
| 信息损失 | 离散化（分词）过程会丢失部分精度 | s1 + s2 层级令牌通过两个粒度补偿精度损失 |
| 预测不确定 | 采样策略引入随机性，每次预测可能不同 | 增大 `sample_count` 取多次平均可降低波动 |
| 计算开销 | Transformer 自回归推理需要逐时间步生成，速度受限 | 提供不同规模模型（mini/small/base/large）供选择 |
| 无法预测突发事件 | 模型基于历史模式，无法预测政策变化、黑天鹅等 | 这是所有基于历史数据的模型的共同局限，非 Kronos 特有 |
| OHLC 逻辑不保证 | 每个价格列独立预测，不保证 high >= max(open, close) 等金融逻辑 | 可通过后处理修复，详见 [错误排查](../references/troubleshooting.md) |

### Kronos 的核心假设

理解以下隐含假设有助于判断 Kronos 是否适合你的任务：

1. **历史模式可延续**：模型假设未来的价格走势在某种程度上会延续历史的统计规律。如果市场发生了结构性变化（如监管政策调整、交易规则修改），模型的历史知识可能失效
2. **离散令牌足以表达 K 线特征**：BSQ 量化将连续的 20 维向量映射为约 100 万种令牌（2^20）。这对大多数 K 线模式足够，但极端行情（如闪崩、乌龙指）可能落在令牌空间的稀疏区域
3. **时间周期性有意义**：TemporalEmbedding 假设"周一"和"下周一"有某种相似性。这在大多数金融市场成立，但在 24/7 市场（如加密货币）中，某些周期性特征的意义会减弱
4. **单只股票的历史足够**：KronosPredictor 使用实例级标准化，每条预测只依赖单只股票的自身历史。它不直接利用跨股票的相关性信息

### Kronos 不适合的场景

| 场景 | 原因 | 推荐替代方案 |
|------|------|------------|
| 高频交易（毫秒级决策） | 推理延迟为秒级 | 传统量化策略 + 低延迟框架 |
| 精确价格预测 | 模型输出是概率性的 | 不存在可靠的精确价格预测方法 |
| 宏观经济分析 | 模型只处理 OHLCV 数据 | 宏观经济模型 + 基本面分析 |
| 非金融时序（天气、传感器等） | 模型针对 K 线特征设计 | TimesFM、Chronos 等通用时序模型 |
| 事件驱动交易 | 模型无法处理新闻、政策等非结构化信息 | 新闻情感分析 + 事件驱动策略 |

## 两阶段框架

```
┌──────────────────────────────────────────────────────────────┐
│                    Kronos 两阶段框架                          │
│                                                              │
│  阶段 1：分词（Tokenization）                                │
│  ┌──────────┐    ┌────────────┐    ┌──────────┐             │
│  │ OHLCV    │ →  │ Encoder    │ →  │ BSQ      │ → 离散令牌  │
│  │ 连续数据 │    │ Transformer│    │ 量化器   │   (s1, s2)  │
│  └──────────┘    └────────────┘    └──────────┘             │
│                                                              │
│  阶段 2：自回归预测（Autoregressive Prediction）             │
│  ┌──────────┐    ┌────────────┐    ┌──────────┐             │
│  │ 历史令牌 │ →  │ Kronos     │ →  │ 采样策略 │ → 预测令牌  │
│  │ (s1, s2) │    │ Transformer│    │ top-k/p  │             │
│  └──────────┘    └────────────┘    └──────────┘             │
│                                       ↓                      │
│                    ┌────────────┐    ┌──────────┐            │
│                    │ OHLCV     │ ←  │ Decoder  │ ← 预测令牌 │
│                    │ 预测数据  │    │ Transformer│            │
│                    └────────────┘    └──────────┘            │
└──────────────────────────────────────────────────────────────┘
```

### 阶段 1：分词（Tokenization）

由 **KronosTokenizer** 完成。它将每根 K 线的 6 维 OHLCV 数据（open、high、low、close、volume、amount）编码为一对离散令牌 `(s1, s2)`：

- **s1（粗粒度令牌）**：捕捉 K 线的主要价格走势方向（[层级令牌体系](05-hierarchical-tokens.md)有详细解析）
- **s2（细粒度令牌）**：在 s1 的基础上提供更精细的修正信息

**类比理解**：s1 就像是"这笔交易大致是涨还是跌"，s2 则是"具体涨跌幅是多少"。s1 和 s2 通过 [DependencyAwareLayer](03-model.md) 实现条件依赖——s2 的预测会"看到" s1 的结果。

### 阶段 2：自回归预测

由 **Kronos** 模型完成。它接收历史令牌序列，逐个预测未来的 s1 和 s2 令牌：

1. 先预测 s1（粗粒度）
2. 基于已预测的 s1，再预测 s2（细粒度）
3. 重复此过程，直到生成指定数量的预测步

最后，分词器的解码器将预测令牌还原为 OHLCV 数值。

## 三大核心 API

Kronos 提供三个面向用户的类，全部通过 `from model import ...` 导入：

### 1. KronosTokenizer（分词器）

**职责**：OHLCV 连续数据 ↔ 离散令牌之间的双向转换

```python
tokenizer = KronosTokenizer.from_pretrained("NeoQuasar/Kronos-Tokenizer-base")

# 编码：OHLCV → 令牌
z_indices = tokenizer.encode(x, half=True)  # 返回 (s1_indices, s2_indices)

# 解码：令牌 → OHLCV
reconstructed = tokenizer.decode(z_indices, half=True)
```

### 2. Kronos（预测模型）

**职责**：给定历史令牌序列，自回归预测未来令牌

```python
model = Kronos.from_pretrained("NeoQuasar/Kronos-small")

# 前向推理（返回 s1 和 s2 的预测 logits）
s1_logits, s2_logits = model(s1_ids, s2_ids, stamp=timestamps)
```

### 3. KronosPredictor（预测器）

**职责**：将分词器 + 模型封装为一步到位的预测接口

```python
predictor = KronosPredictor(model, tokenizer, max_context=512)

# 一键预测
pred_df = predictor.predict(df=x_df, x_timestamp=..., y_timestamp=..., pred_len=120)
```

**推荐用法**：日常使用中直接使用 `KronosPredictor`，只有在微调或研究时才需要单独操作 `KronosTokenizer` 和 `Kronos`。

## 模型动物园

Kronos 提供不同规模的预训练模型，均托管在 HuggingFace Hub 的 `NeoQuasar` 组织下：

| 模型 | Hub 路径 | 参数量 | 特点 |
|------|----------|--------|------|
| Kronos-Tokenizer-2k | `NeoQuasar/Kronos-Tokenizer-2k` | — | mini 专用分词器（上下文 2048） |
| Kronos-Tokenizer-base | `NeoQuasar/Kronos-Tokenizer-base` | — | small/base/large 共用分词器（上下文 512） |
| Kronos-mini | `NeoQuasar/Kronos-mini` | 4.1M | 最小模型，推理最快，上下文 2048 |
| Kronos-small | `NeoQuasar/Kronos-small` | 24.7M | 速度与效果的平衡点，推荐入门使用 |
| Kronos-base | `NeoQuasar/Kronos-base` | 102.3M | 标准模型 |
| Kronos-large | `NeoQuasar/Kronos-large` | 499.2M | 最大模型（未开源） |

> **注意**：Kronos-mini 使用专用的 `Kronos-Tokenizer-2k` 分词器，其余三个模型共用 `Kronos-Tokenizer-base`。在 small/base/large 之间切换无需更换分词器，但涉及 mini 时需要同时更换。详见 [模型对比与选型指南](../advanced/07-model-comparison.md)。

## 支持的市场与时间粒度

根目录 `README.md` 明确写到项目在 45+ 个全球交易所数据上进行了预训练。结合仓库中的示例脚本，可以确认文档覆盖了以下几类用法：

- **不同市场来源**：通用 CSV、A 股示例、加密货币 Web UI 示例
- **不同时间粒度**：分钟线和日线在仓库示例中都出现过
- **统一输入结构**：预测接口要求使用固定列名的 OHLCV/Amount 数据框

需要注意的是，”预训练覆盖很多市场”不等于”对任何市场都一定有效”。对于特定市场、频率或策略目标，仍然需要用你自己的数据做回测或验证。

### 不同时间粒度的推荐配置

| 时间粒度 | 推荐 lookback | 推荐 pred_len | 说明 |
|---------|-------------|-------------|------|
| 1 分钟线 | 200-400 | 30-60 | 超短线分析，上下文约 3-7 小时 |
| 5 分钟线 | 200-400 | 60-120 | 日内分析，上下文约 2-4 天 |
| 15 分钟线 | 200-400 | 40-80 | 日内趋势，上下文约 2-4 天 |
| 1 小时线 | 200-512 | 24-72 | 短期趋势，上下文约 1-3 周 |
| 日线 | 200-512 | 20-60 | 中期趋势，上下文约 1-2 年 |
| 周线 | 200-512 | 12-24 | 长期趋势，上下文约 4-10 年 |
